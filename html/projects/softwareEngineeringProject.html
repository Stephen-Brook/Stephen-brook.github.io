Software Engineering Project

About {

This project focused on waterfall software development. The project entailed meeting with steakholders, creating requirements, class a diagram, sequence diagrams, creating a test plan, and finally, creating the product.

The end product was a grading calculator, designed specifically to meet the steakholders requirements.

}

Stage 1 (Requirements) {

This stage involved meeting with steak holders to discuss their exact requirements of the system. Those requirements were:

  • The user can input multiple categories, and assignments within those categories
  • The user can input multiple courses
  • The user can reload previously saved data
  • The user can input 'predicted' grades, and calculate how that will affect their overeall grade in a class
  • The user can dictate rounding criteria

}

Stage 2 (Class Diagram) {

This stage focused on creating a class diagram that would dictate how the project would function. The class diagram was required to follow the steakholders requirements.

}

Stage 3 (Sequence Diagrams) {

This stage focused on creating a sequence diagrams that would dictate the flow of events for a given requirement. The sequence diagrams were required to follow the steakholders requirements.

}
